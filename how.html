How Do We Apply Universal Design?

There are many ways that we can possibly get involved let's look at a couple of popular options!

Education and Training

Start by educating yourself on web accessibility standards and guidelines such as WCAG (Web Content Accessibility Guidelines). There are numerous online resources, courses, and certifications available that can help you understand the principles and techniques required for creating accessible websites.

Join Accessibility Communities and Forums

Engage with online communities, forums, or groups dedicated to web accessibility. Participating in discussions, sharing knowledge, and seeking advice from experts can help you learn and network within the accessibility community.

Advocate for Accessibility in your Workplace

If you're already working in a web development role, advocate for accessibility within your company. Encourage the adoption of accessibility best practices in ongoing projects and suggest making existing websites more accessible.

Using Code That Promotes Accessibility

Semantic HTML

Using semantic HTML means organizing website content in a smart way. It helps computers and tools understand what each part of the content is about, making it easier for everyone to use the website. It also helps with how well the website appears in search results. When we organize content this way, it also makes it easier for web developers to work with and keep things in order. So, it's like arranging things neatly for everyone to understand easily, both for people and computers.

Below is an example of what a snippet of some semantic code may look like:

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <title>Semantic HTML Example</title> </head> <body> <header> <h1>Welcome to My Website</h1> </header> <main> <section> <h2>About</h2> <p>Learn more...</p> </section> </main> <footer> <p>&copy; 2023 My Website</p> </footer> </body> </html>

Skip Navigation

Skip navigation is helpful because it allows people using screen readers or keyboards to skip repetitive content on a webpage, like menus, and go straight to the main content. This makes the website more accessible and saves time for those who navigate differently, ensuring they can easily find what they need without going through every menu each time they visit a page.

Below is an example of what a snippet of some skip nav code may look like:

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <title>Skip Navigation Example</title> </head> <body> <a href="#main-content" style="position:absolute; left:-9999px;">Skip to main content</a> <header> <h1>Welcome to My Website</h1> <!-- Navigation and other content --> </header> <main id="main-content"> <!-- Main content --> </main> <footer> <p>&copy; 2023 My Website</p> </footer> </body> </html>

Screen Reader Accessible

Screen-reader-only code is useful because it provides information specifically for users who rely on screen readers to access websites. It includes hidden text or descriptions that are only heard by screen readers, ensuring that visually impaired users get important information such as alternative text for images or descriptions of visual elements, making the website more accessible to them.

Below is an example of what a snippet of some screen-reader only code may look like:

<div aria-hidden="true">This text is for screen readers only</div>
